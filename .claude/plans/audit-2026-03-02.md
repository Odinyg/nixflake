# NixOS Config Audit Report — 2026-03-02

**Scanned:** 56 home-manager modules, 26 NixOS modules, 5 profiles, 3 hosts
**Issues found:** 2 critical, 7 high, 11 medium, 8 low

---

## CRITICAL

### 1. [modules/nixos/work/communication.nix:4] Dead `nixpkgs-stable` reference
References `inputs.nixpkgs-stable` which does not exist in `flake.nix`. The `let pkgs-stable = import inputs.nixpkgs-stable ...` binding is currently unused (packages use `pkgs`), so it doesn't error yet — but will break the moment someone uses `pkgs-stable`.

**Fix:** Remove the unused `pkgs-stable` let-binding.

### 2. [profiles/base.nix:122] fail2ban unconditionally enabled
`services.fail2ban.enable = true` is set unconditionally at line 122, alongside `fail2ban-security.enable = lib.mkDefault true` at line 76. When `hosts/laptop/default.nix` overrides `fail2ban-security.enable = false`, the custom module's config is guarded off — but `services.fail2ban.enable = true` still fires, keeping fail2ban active on laptop.

**Fix:** Remove `services.fail2ban.enable = true` from line 122 (the `fail2ban-security` module already handles this).

## HIGH

### 1. [profiles/hardware/nvidia.nix:55] Bogus WLR_DRM_DEVICES path — REMOVING
`WLR_DRM_DEVICES = "$HOME/.config/hypr/card:$HOME/.config/hypr/otherCard"` — placeholder paths from a wiki guide. No host or module creates these files. Only referenced in this one location. Causes wlroots to look for non-existent files.

**Fix:** Remove this environment variable entirely (let wlroots auto-detect).

### 2. [modules/nixos/ollama.nix:7] Ollama binds to 0.0.0.0 — INTENTIONAL
`host = "0.0.0.0"` with `openFirewall = true`. Ollama is intentionally exposed to the LAN for access from other machines. **This is a deliberate design choice — do not flag again.**

Added to CLAUDE.md as a known intentional configuration.

### 3. [modules/nixos/syncthing.nix:16-18] overrideDevices with empty devices
`overrideDevices = true` with `devices = { }` actively wipes any devices added through the WebUI on every rebuild.

**Fix:** Remove `overrideDevices = true` and `settings.devices = { }` so WebUI-managed devices persist.

### 4. [profiles/hardware/nvidia.nix:47-49] PRIME sync wrong for station
PRIME sync is always enabled with Intel bus ID defaults. Station is a desktop with AMD CPU + NVIDIA GPU (no Intel iGPU). PRIME sync on a desktop-only GPU is wrong. Station should use `modules/nixos/hardware/nvidia-gpu.nix` (desktop-optimized module) instead.

**Fix:** Station already has `hardware.nvidia-gpu.enable = true` in its host config AND imports `profiles/hardware/nvidia.nix`. The profile's PRIME config conflicts. Remove `profiles/hardware/nvidia.nix` from station's imports — the desktop nvidia-gpu module is sufficient.

### 5. [modules/home-manager/misc/gammastep.nix:25] randr on Wayland
`adjustment-method = "randr"` is X11-only. On Hyprland (Wayland), this silently fails.

**Fix:** Change to `adjustment-method = "wayland"`.

### 6. [modules/home-manager/desktop/hyprland/keybindings.nix:17] Wrong browser binary
`exec, zen-beta` — The binary is called `zen`, not `zen-beta`.

**Fix:** Change to `exec, zen`.

### 7. [modules/home-manager/desktop/hyprland/default.nix:22] Cursor name typo
`hyprctl setcursor Bibate-Modern-Ice 18` — Typo: "Bibate" not "Bibata". Confirmed by `styling.nix:50` and `hosts/laptop/default.nix:95` which both use "Bibata".

**Fix:** Change to `hyprctl setcursor Bibata-Modern-Ice 18`.
