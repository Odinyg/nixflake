# Add secrets for a server service

Generate secrets and add them to encrypted `secrets/<host>.yaml`. Works in two modes:
1. **Plan-based:** reads secrets from `plans/<service>.md`
2. **Module-based:** scans `modules/server/<service>.nix` for `sops.secrets.*` and `sops.templates.*` references

## Step 1: Parse arguments

Parse `$ARGUMENTS` for the service name and optional host (e.g. `/server-add-secrets mealie sugar`).
If service name is missing, ask the user.

## Step 2: Discover required secrets

Try sources in order until one works:

### 2a: Plan file
Look in `plans/<service>.md` or `plans/archive/<service>.md`.
If found, extract secrets from the `secrets/<host>.yaml` REMINDER section and `sops.secrets.*` / `sops.placeholder.*` references.
Read `**Host:** <host>` from the plan header.

### 2b: NixOS module
Read `modules/server/<service>.nix`. Extract all secret names from:
- `sops.secrets.<name>` declarations
- `config.sops.placeholder.<name>` references in templates
- `sops.templates."<name>"` declarations

If host wasn't provided as argument, find which host enables the service:
```bash
grep -l "server.<service>" hosts/*/default.nix
```

### 2c: Host config
If neither plan nor module exists, scan `hosts/<host>/default.nix` for `sops.secrets.*` references related to the service name.

Build a list of `(secret_name, description)` pairs.

## Step 3: Categorize each secret

### Category A: Auto-generate (passwords, keys, generic secrets)

| Pattern | Generation method |
|---------|-------------------|
| `*encryption_key*`, `*master_key*`, `*jwt_secret*`, `*session_secret*`, `*signing_key*`, `*secret_key*` | `openssl rand -hex 32` (64 hex chars) |
| `*_pass`, `*_password` | `openssl rand -base64 32` |
| `*_api_key*` | `openssl rand -hex 24` (48 hex chars) |

**Note:** `postgresql_<service>_password` secrets are auto-generated by the `server.postgresql` module via `sops.secrets`. They still need a value in SOPS — generate with `openssl rand -base64 32`.

### Category B: OIDC client secrets (auto-generate with Authelia)

Secret names matching `*oidc_client_secret*` get special handling:

1. Generate a random plaintext password and its Authelia pbkdf2-sha512 hash:
```bash
nix-shell -p authelia --run "authelia crypto hash generate pbkdf2 --variant sha512 --random"
```

2. This outputs two lines:
   - `Random Password: <plaintext>` — this goes into SOPS as the secret value
   - `Digest: $pbkdf2-sha512$...` — this goes into the Authelia config

3. Parse both values. Store the **plaintext password** in SOPS. Display the **digest hash** and tell the user it needs to go into `modules/server/authelia.nix` as the `client_secret` for the service's OIDC client.

### Category C: Ask the user
- API keys/tokens from external services
- Email addresses, URLs
- Anything that can't be auto-generated

## Step 4: Check for existing secrets

Before generating, check what already exists:
```bash
sops -d secrets/<host>.yaml 2>/dev/null | grep -E '^(secret_name_1|secret_name_2):' || true
```

For each secret that already exists, warn and skip unless the user explicitly asks to regenerate.

## Step 5: Generate and collect values

1. **Auto-generate** all Category A values
2. **Generate OIDC** Category B values via `nix-shell -p authelia`
3. **Ask the user** for all Category C values

Show a summary:
```
Auto-generated:
  postgresql_mealie_password = (random base64)
  mealie_secret_key = (random 64 hex)

OIDC (auto-generated):
  mealie_oidc_client_secret = (random - will also output Authelia digest hash)

Need your input:
  some_external_api_key = ?
```

## Step 6: Write to secrets/<host>.yaml

Use `sops --set` for each secret (non-interactive, no temp files):

```bash
sops --set '["<secret_name>"] "<value>"' secrets/<host>.yaml
```

If the file doesn't exist yet, create it first:
```bash
sops -e /dev/null > secrets/<host>.yaml
```
Then use `sops --set` as normal.

## Step 7: Summary

Show the user:

**Secrets added to `secrets/<host>.yaml`:**
- `secret_name` — auto-generated / oidc / user-provided

If OIDC secrets were generated, show:

**Authelia OIDC digest (add to `modules/server/authelia.nix`):**
```
client_secret = "$pbkdf2-sha512$310000$...";
```

**Next steps:**
- Verify with `sops secrets/<host>.yaml` (opens decrypted in editor)
- Deploy: `colmena apply --on <host>`
